# クラウドコスト管理ルール

## 目的
Azure 利用コストの最適化と予算超過の防止を目的とします。

## 予算管理

### 予算上限
| 環境 | 月額上限 | アラート閾値 |
|------|----------|-------------|
| 本番 | プロジェクトごとに設定 | 80%, 100%, 120% |
| 開発 | 50,000 円/プロジェクト | 80%, 100% |
| テスト | 30,000 円/プロジェクト | 80%, 100% |
| 個人検証 | 10,000 円/人 | 80%, 100% |

### 予算アラート
- Azure Cost Management でプロジェクトごとの予算を設定すること
- アラートは管理責任者およびクラウド推進チームに通知

## SKU 選定基準

### App Service / Functions
| 用途 | 推奨 SKU |
|------|----------|
| 本番 Web アプリ | P1v3 以上（自動スケール対応） |
| 本番 API | P1v3 または Functions Premium |
| 開発・テスト | B1（Basic）または F1（Free） |
| バッチ処理 | Functions Consumption（従量課金） |

### データベース
| 用途 | 推奨 |
|------|------|
| 小規模 | Azure SQL Database Basic / S0 |
| 中規模 | Azure SQL Database S1-S3 |
| 大規模 | Azure SQL Database Hyperscale |

### AI サービス
| サービス | 推奨 SKU |
|----------|----------|
| Azure OpenAI | S0（リクエスト数に応じて TPM を調整） |
| AI Search | Free（検証用）/ Basic（本番用） |
| Cognitive Services | S0 |

## コスト最適化ルール

### 自動スケール
- 本番環境の App Service は自動スケールを設定すること
- 最小インスタンス数: 2（可用性確保）
- 最大インスタンス数: プロジェクトの予算に応じて設定

### 開発・テスト環境の運用
- 業務時間外（20:00-08:00）は開発・テスト環境のリソースを停止すること
- Azure Automation または Logic Apps で自動停止/起動を設定
- 未使用リソースは月次で棚卸しし、不要なものは削除

### リザーブドインスタンス
- 1 年以上継続利用が見込まれるリソースはリザーブドインスタンスを検討（最大 72% 割引）
- 購入前にクラウド推進チームに相談すること
