# クラウドセキュリティポリシー

## 目的
本ドキュメントは、Azure 上のリソースに適用するセキュリティ要件を定めるものです。

## 認証・認可

### マネージド ID の利用（必須）
- Azure リソース間の認証には**システム割り当てマネージド ID** を使用すること
- API キーやパスワードによる認証は原則禁止
- やむを得ず API キーを使用する場合は Key Vault に格納し、定期的にローテーションすること（90日以内）

### RBAC（ロールベースアクセス制御）
- 最小権限の原則に従い、必要最低限のロールのみを割り当てること
- `Owner` ロールの割り当てはクラウド推進チームの承認が必要
- カスタムロールの作成は原則禁止（ビルトインロールを使用）
- サービスプリンシパルへの権限付与は期限付きとし、最大 1 年で見直すこと

## ネットワークセキュリティ

### パブリックアクセス制限
- 本番環境のリソースは原則パブリックアクセスを無効化すること
- App Service、Functions は VNet 統合または Private Endpoint を使用
- Storage Account は `publicNetworkAccess: disabled` に設定

### 例外
- 開発・テスト環境では IP 制限付きでパブリックアクセスを許可可能
- ただし、`0.0.0.0/0` の許可は全環境で禁止

## データ保護

### 暗号化
- 保存データ: Azure 既定の暗号化（SSE）を使用。カスタマーマネージドキーは任意
- 転送データ: TLS 1.2 以上を必須とする
- Storage Account は `minimumTlsVersion: TLS1_2` を設定

### 機密データの取り扱い
- 個人情報（PII）を含むデータは日本リージョンにのみ保存すること
- ログに機密情報を出力しないよう Application Insights のフィルタリングを設定

## 監査・モニタリング
- すべてのサブスクリプションで Azure Monitor と Defender for Cloud を有効化
- 診断ログは Log Analytics ワークスペースに集約（保持期間: 90日以上）
- セキュリティアラートは Slack の `#security-alerts` チャンネルに通知
